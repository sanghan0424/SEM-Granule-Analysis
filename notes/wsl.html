二、WSL2 基础命令学习

1. 核心命令：cd 与 ls

  • cd 路径：切换目录。
    • cd /home/你的用户名/：进入用户主目录。
    • cd SEM-Granule-Analysis：进入项目文件夹。
    • cd ..：返回上一级目录。
    • cd ~：快速回到用户主目录。
  • ls：查看当前目录下的文件与文件夹。
    • ls -l：显示详细信息（权限、大小、修改时间）。
    • ls /mnt/d/：查看 Windows D 盘内容。
💡 路径小技巧：WSL2 中 Windows 磁盘被挂载到 /mnt/ 目录下，例如 D:\ 对应 /mnt/d/。

2. 访问 Windows 项目文件夹

  1. 进入 D 盘：cd /mnt/d/
  2. 查看 D 盘内容：ls
  3. 进入项目目录：cd /mnt/d/SEM-Granule-Analysis/
  4. 验证当前路径：pwd（输出当前目录的完整路径）

三、Git 克隆 GitHub 仓库

1. 前置检查

  • 检查 Git 是否安装：git --version。
  • 未安装则执行：sudo apt update && sudo apt install git -y。

2. 克隆命令

在项目目录下执行（替换为你的仓库地址）：

git clone https://github.com/你的用户名/SEM-Granule-Analysis.git .
  • 末尾的 . 表示将仓库克隆到当前目录，避免嵌套。
  • 若提示登录，使用 GitHub 个人访问令牌（PAT）作为密码。

3. 验证克隆结果

  • 进入克隆后的目录：cd SEM-Granule-Analysis
  • 查看文件：ls，确认能看到 README.md。

四、WSL2 文件系统性能优化

⚠️ 性能提示：在 Windows 驱动器（/mnt/d/）上执行 Git 等 I/O 密集操作会有性能损耗。

1. 项目迁移到 Linux 文件系统（推荐）

  1. 创建 Linux 项目目录：mkdir -p ~/SEM-Granule-Analysis
  2. 复制 Windows 文件到 Linux：cp -r /mnt/d/SEM-Granule-Analysis/* ~/SEM-Granule-Analysis/
  3. 进入 Linux 项目目录：cd ~/SEM-Granule-Analysis
  4. 在 Linux 目录下执行 Git 操作，性能更佳。

2. Windows 访问 Linux 文件

在 Windows 文件管理器地址栏输入：

\\wsl$\Ubuntu\home\你的用户名\SEM-Granule-Analysis

即可直接访问 WSL 内的项目文件，实现双向同步。

五、今日问题与解决

问题 解决方法
WSL 提示 Windows 驱动器 I/O 性能差 将项目迁移到 Linux 目录 ~/SEM-Granule-Analysis
Git clone 提示权限拒绝 使用 GitHub 个人访问令牌（PAT）作为密码
路径区分大小写导致 cd 失败 确保 WSL 路径与 Windows 文件夹名大小写完全一致
